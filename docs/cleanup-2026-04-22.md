## B. 結構不一致（4 項，建議重組）

### B1. `markets` 沒有獨立 blueprint
**現況**：
- 有 `templates/markets/index.html` + `static/css/markets.css` + `static/js/markets.js`
- 但路由 `/markets` 塞在 [blueprints/stock/routes.py:652](../Aurum_Infinity_AI/blueprints/stock/routes.py#L652)，沒有獨立 `blueprints/markets/` 目錄
**影響**：違反現有的「一頁一 blueprint」慣例（news_radar、insider、ipo、trending 都獨立）
**建議**：抽出 `blueprints/markets/`，把 markets 相關 route 搬過去

### B2. `services/` 沒有子資料夾分類
**現況**：9 個 service 檔案平鋪：
- `gemini_service.py`（LLM 基礎）
- `feature_article_service.py`、`news_service.py`、`seeking_alpha_report_service.py`、`radar_topics_service.py`（新聞類）
- `ipo_store.py`、`insider_service.py`（資料存取類）
- `data_vars.py`（prompt 變數解析）
**建議**：
```
services/
├── llm/         # gemini_service.py, data_vars.py
├── news/        # news_service, feature_article, seeking_alpha_report, radar_topics
└── data/        # ipo_store, insider_service
```
**注意**：搬動會動到 import path，屬於中度改動，建議列為 Phase 2 優先

### B3. `News_features/` vs `News_fetcher/` 命名易混淆
**現況**：
- `News_features/` — 手寫專題 HTML（IPO 深度報告、戰爭影響分析等）
- `News_fetcher/` — 自動抓新聞的 Python 腳本 + futunn 快取
兩個資料夾用途完全不同，但名字只差一個字
**建議**：改名 `News_features/ → Feature_articles/`，或用 `docs/special_reports/`
**風險**：有 manifest.json 指向這些 HTML，改名要同步更新 reference

### B4. 根目錄有 `Get_stock/` 疑似舊模組
**位置**：[Get_stock/](../Get_stock/)
**現況**：跟 `Aurum_Infinity_AI/stock_code/` 和 `Aurum_Data_Fetcher/ticker.py` 都有功能重疊
**建議**：需你判斷 `Get_stock/` 是否還在用；若否，搬到 `_archive/` 或刪除

## D. 架構警訊（不刪，但建議改造）

### D1. blueprint 之間透過 `from blueprints.stock.routes import ...` 互相依賴
**現況**：
- `ipo/routes.py:15`、`news/routes.py:6`、`insider/routes.py:4`、`trending/routes.py:7` 都 `from blueprints.stock.routes import get_current_lang`
- `admin/routes.py:48` `from blueprints.stock.routes import resolve_ticker, get_today`
**問題**：stock blueprint 變成隱性的 core module，換 stock blueprint 會牽動所有 blueprint
**建議**：把 `get_current_lang`、`resolve_ticker`、`get_today`、`is_valid_ticker` 抽到 `utils/request_helpers.py` 或 `utils/ticker_helpers.py`

### D2. `blueprints/stock/routes.py` 過大（1566 行）
**現況**：同一個 routes.py 同時處理：
- `/<ticker>` 股票頁
- `/markets` 市場頁
- `/api/rating_verdict`、`/api/analyze_section`、`/api/price_analysis`（AI 端點）
- ticker 解析、評分、cache key、helper function
**建議**：拆為 `routes/main.py`（頁面）+ `routes/api.py`（AI 端點）+ `helpers.py`（工具函式）

### D3. 測試檔分布不一致
**現況**：
- `Aurum_Infinity_AI/tests/` 同時放了 Flask blueprint 測試和 data fetcher 測試（`test_data_fetcher_main.py`、`test_schema_validator.py`、`test_refresh_news_cache.py`）
**問題**：`test_data_fetcher_main.py` 測的是 `Aurum_Data_Fetcher/` 程式碼，但放在 `Aurum_Infinity_AI/tests/` 下
**建議**：把 fetcher / schema / news_fetcher 相關測試搬到各自的 `tests/` 目錄
